## 📋 Overview

This project implements a **shell-based Intrusion Detection System (IDS)** that analyzes raw authentication logs, identifies brute-force attackers, and generates firewall rules to block malicious IP addresses. The system is built entirely using core Unix utilities — `sed`, `awk`, `grep`, and shell loops.

## 🚀 Scripts

### Task 1: `sanitize.sh` — The Log Sanitizer

Cleans the raw `auth.log` by applying a `sed` pipeline:

| Step | Operation | Details |
|------|-----------|---------|
| 1 | **Remove corruption** | Deletes lines containing `[CORRUPT-DATA]` |
| 2 | **Anonymize users** | Replaces `user=root` and `user=admin` → `user=SYS_ADMIN` |
| 3 | **Standardize delimiters** | Converts pipe (`\|`) separators to commas (`,`) |
| 4 | **Strip carriage returns** | Removes Windows `\r` characters via `tr` |

**Usage:**
```bash
bash sanitize.sh auth.log
```

**Output:** `clean_log.csv`

---

### Task 2: `detect.sh` — The Brute-Force Detector

Identifies malicious IPs from the cleaned log and generates `iptables` firewall rules.

**Pipeline:**
1. **`awk`** parses `clean_log.csv` to count `Failed password` events per IP.
2. **Threshold filter** — only IPs with **> 10** failed attempts are flagged.
3. **Whitelist cross-reference** — suspect IPs are checked against `whitelist.txt` using a manual shell loop (no `grep -f`, `comm`, or `diff` allowed).
4. **Rule generation** — non-whitelisted IPs are written as `iptables DROP` rules.

**Usage:**
```bash
bash detect.sh clean_log.csv whitelist.txt firewall_rules.sh
```

**Output:** `firewall_rules.sh` — example contents:
```
iptables -A INPUT -s 192.168.1.100 -j DROP # Blocked after 12 failed attempts
iptables -A INPUT -s 45.33.22.11 -j DROP # Blocked after 11 failed attempts
```

---

### Task 3: `report.sh` — The Report Dashboard

Generates a summary of targeted ports across all failed login attempts.

**Usage:**
```bash

```

**Sample Output:**
```
Target Port Analysis
-------------------

Port 8080 : 10 attempts
Port 443  : 3 attempts
```

### Task 4: `timeline.sh` — The Threat Timeline

Displays failed login attempts aggregated by hour (00–23) to reveal attack patterns.

**Usage:**
```bash

```

**Sample Output:**
```
Hour 09: 21 failed attempts
Hour 10: 9 failed attempts
Hour 11: 11 failed attempts
```

## 🧰 Tools & Techniques Used

- **`sed`** — Stream editing for log sanitization (pattern deletion, substitution)
- **`awk`** — Field-based text processing for log parsing and aggregation
- **`tr`** — Character translation (carriage return removal)
- **`sort`** — Sorting report output by attempt count
- **Shell loops** — Manual whitelist cross-referencing (per assignment constraints)
- **`printf` / `sprintf`** — Formatted output with zero-padded hours

---

## 📝 Assignment Constraints

- Whitelist filtering in `detect.sh` must use manual shell loops — `grep -f`, `comm`, and `diff` are **not allowed**.
- Brute-force threshold is **strictly greater than 10** failed attempts.
- Output formats must match the exact specifications from the problem statement.
